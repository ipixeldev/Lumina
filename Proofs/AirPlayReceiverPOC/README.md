# AirPlay Receiver Proof

This standalone macOS target validates one narrow milestone: a real iPhone should discover a separate receiver named `Lumina – <Mac name>` in Control Center → Screen Mirroring.

It intentionally contains only:

- a persistent experimental receiver identity;
- `_airplay._tcp` and `_raop._tcp` Bonjour publication;
- a bounded TCP listener;
- a visible event log for the sender's first request;
- an explicit `501 Not Implemented` response.

It does not contain AirPlay authentication, FairPlay, H.264 reception, decoding, audio, screen capture, XCTest control, or any production Lumina integration. Selecting the receiver is expected to stop after the proof records the first request.

## Verified result

On 20 July 2026, the proof appeared as a separate `Lumina – <Mac name>` destination on the physical iPhone 15 Plus running iOS 26.1. Selecting it reached the proof listener and then showed the expected connection failure because authentication is deliberately not implemented. This validates only discovery and first-request routing, not a working media receiver.

## Build and run

The generated Xcode project uses the repository's existing `Configuration/Signing.xcconfig`. Copy the local signing example if the repository has not already been configured:

```bash
cp Configuration/LocalSigning.xcconfig.example Configuration/LocalSigning.xcconfig
```

Set your Apple Development Team ID in that ignored local file, then build:

```bash
xcodebuild \
  -project Proofs/AirPlayReceiverPOC/AirPlayReceiverPOC.xcodeproj \
  -scheme AirPlayReceiverPOC \
  -configuration Debug \
  -destination 'platform=macOS,arch=arm64' \
  build
```

Open `Proofs/AirPlayReceiverPOC/AirPlayReceiverPOC.xcodeproj` and run the shared `AirPlayReceiverPOC` scheme. Accept the one-time Local Network permission if macOS asks.

On the iPhone, open Control Center → Screen Mirroring and look for the receiver name shown by the proof. The Mac and iPhone must be on the same local network.

You can independently inspect the advertisement from Terminal:

```bash
dns-sd -B _airplay._tcp local.
dns-sd -B _raop._tcp local.
```

## Regenerate the project

The checked-in Xcode project is generated by `make_project.rb` using Ruby `xcodeproj` 1.27.0. The generator pins that exact version because its deterministic object-ID handling is version-sensitive:

```bash
cd Proofs/AirPlayReceiverPOC
ruby make_project.rb
```

The production `Lumina` target has no dependency on this proof.
